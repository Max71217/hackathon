# 需求分析与问题分解

## 1. 问题背景
在现代医学教育与临床诊断中，医学知识体系极其庞大且分散。例如，“心力衰竭”相关的生理学基础、病理学机制、解剖学定位以及药理学治疗，往往分布在数本高达数百页的不同教材中。学生和医生在查阅时面临严重的“知识孤岛”问题，难以快速建立跨学科的立体关联网络。

## 2. 核心挑战
本系统在对 7 本总计约 826MB 的医学教材进行整合时，面临三大核心挑战：
1. **多模态图文混排**：医学教材包含大量的解剖图谱、病理切片和生理流程图。纯文本解析会彻底丢失图像中的空间与逻辑信息。
2. **实体歧义与冗余**：不同教材对同一医学概念的表述往往存在差异（如《病理学》的“心衰”与《生理学》的“心力衰竭”），强行整合易造成节点爆炸或关联断裂。
3. **严格的逻辑前置性**：医学知识具备极强的因果与依赖关系，传统切片式 RAG（检索增强生成）容易破坏上下文逻辑，导致问答“幻觉”。

## 3. 子问题分解与解决思路

### 子问题一：如何高效处理海量多模态 PDF 数据？
*   **分析**：如果将 800MB 教材全部交由多模态大模型处理，将导致严重的 API 并发阻塞及成本失控；全盘使用 OCR 则无法理解复杂插图。
*   **解法**：建立**基于阈值的智能感知分流（Smart Routing）**。以 400 字符为界，高密度文本页走快速文本模型，低密度/插图页走强视觉理解模型。

### 子问题二：如何统一结构化知识的提取标准？
*   **分析**：大模型自由生成的 JSON 格式极易失控，导致前后端数据链断裂。
*   **解法**：引入“契约驱动开发（Contract-Driven Development）”。使用 Pydantic 确立唯一的 `schema.py` 数据宪法，强制定义 `ConceptNode`（节点）和 `RelationEdge`（边，限定为前置、包含、并列等），并通过工业级正则约束大模型的输出。

### 子问题三：如何跨教材进行图谱融合与消歧？
*   **分析**：在整合两本甚至多本教材时，算法不仅要合并相似概念，还必须使融合后的描述字数压缩率 < 30%，且绝对不能弄断原有的知识关联（Edges）。
*   **解法**：放弃低效的传统 NLP 相似度计算算法，直接采用拥有长文本和高阶推理能力的大模型进行**端到端融合（End-to-End Merging）**，在脑内进行本体对齐，输出保留多重 `textbook_source` 来源的全新图谱 JSON。

### 子问题四：如何利用图谱提供精准医疗问答？
*   **分析**：单纯的文本 RAG 无法回答需要“跨节点推理”的医学问题。
*   **解法**：构建 **Graph-RAG** 引擎。将结构化的节点定义和边关系转化为自然语言上下文，引导大模型顺着知识连线进行有理有据的推理问答。
