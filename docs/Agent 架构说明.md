# Agent 架构设计决策与论证

## 1. 架构设计总览 (Architecture Overview)
面对 7 本总计约 826MB 的多模态医学教材，传统的单体大模型或纯文本 RAG 架构无法兼顾“解析速度”、“视觉图表理解”与“复杂逻辑推理”。

因此，本项目摒弃了冗余的传统多智能体对话框架，创新性地采用了一套务实且高效的 **“混合多大脑协同架构 (Hybrid Multi-Brain Synergy)”**。整个系统通过严格的数据契约（Schema）解耦，分为“感知”、“逻辑”与“服务”三层，在极短的响应时间内实现了跨医学教材的知识抽取与图谱对齐。

---

## 2. 核心大脑选型与论证 (Brain Selection & Argumentation)

在 5 小时的极限开发中，我们通过灰度测试排除了部分存在严苛医疗图像安全拦截（Safety Filter）的模型，最终确立了以下“双脑协同”选型：

### 2.1 感知大脑 (Perception Agent)：智谱 GLM 系列
- **选型**：`glm-4-flash` (处理纯文本) + `glm-4.6v` (处理视觉图表)。
- **论证**：医学教材中包含大量骨骼、解剖等极易触发大模型安全策略的图片。在实测中，智谱大模型对学术类医学图谱展现出了极佳的宽容度与顶级的 OCR 结构化提取能力，保障了图谱生成的稳定性。

### 2.2 逻辑大脑 (Logic Agent)：DeepSeek-V4-Pro
- **选型**：DeepSeek 官方最新旗舰模型 `deepseek-v4-pro`。
- **论证**：跨教材图谱整合（Ontology Alignment）是本项目最大的难点。DeepSeek-V4-Pro 具备高达 384K 的超长上下文窗口以及原生的 `json_object` 输出能力。它能够在不借助复杂 Python 规则代码的情况下，直接通过强大的逻辑推理完成两个图谱 JSON 间的实体消歧与关系重构。

## 3. 关键架构决策 (Key Architectural Decisions)

本项目能稳定处理多本复杂教材，得益于以下三项核心工程决策：

### 决策一：基于动态字符阈值的“智能路由解析” (Smart Routing)
- **挑战**：800MB 教材如果全部走视觉多模态模型，API 调用耗时与成本将呈指数级爆炸；如果全走本地文本提取，将彻底丢失解剖插图中的空间与逻辑关联。
- **决策**：在 `pdf_parser.py` 中引入动态路由机制。通过 PyMuPDF 试探性提取当前页文本，设定 `TEXT_THRESHOLD = 400`。
  - 若有效字符 `> 400`，判定为**文本页**，交由 Flash 模型极速处理。
  - 若有效字符 `<= 400`，判定为**插图/扫描页**，将其转化为图像字节流，交由多模态视觉模型（Vision）处理。
- **论证**：该机制实现了速度与精度的完美平衡，是系统兼顾高性能与多模态解析的核心保障。

### 决策二：端到端知识图谱对齐 (End-to-End Graph Merging)
- **挑战**：不同教材对同一概念的表述存在差异（如《局部解剖学》与《病理学》中的“下颌骨”）。传统做法需手写 TF-IDF 或向量相似度算法进行遍历比对，极易产生合并冲突。
- **决策**：将图谱 A 和图谱 B 的完整 JSON 并行喂给 DeepSeek-V4-Pro，利用 Prompt 约束其进行“端到端”融合，直接输出一个带有双重 `textbook_source` 出处且继承所有 `edges` 关联的全新 JSON 图谱。
- **论证**：充分利用了大模型的高阶涌现能力（Emergent Abilities），将繁琐的“算法级合并”降维打击为“语义级合并”，在保证压缩比 < 30% 的同时，绝对捍卫了医学逻辑的连贯性。

### 决策三：契约驱动与柔性容错机制 (Resilient Fallback)
- **挑战**：大模型的 JSON 输出不可控，且单页解析报错（如遇到极其复杂的表格无法解析）极易导致数百页的文档处理任务全盘崩溃。
- **决策**：
  1. **契约锁死**：建立严谨的 Pydantic `schema.py` 数据模型，作为系统的唯一交互宪法。
  2. **工业级正则提纯**：在底层引擎中加入 `re.search(r'\{.*\}')` 暴力提取器，无视大模型的任何前置/后置“废话”。
  3. **非阻塞异常降级**：在循环解析流程中捕获异常，若某页彻底失败，仅在终端打印 Debug 日志，并在 UI 右下角弹出非阻塞的 `st.toast` 警告，随后 `continue` 执行下一页，绝不使用 `st.stop()` 中断主进程。
- **论证**：这是保障黑客松演示（Demo）高可用性的最高指导原则。它确保了哪怕存在局部坏页，系统依然能震撼地渲染出 99% 的知识图谱。

---

## 4. 创新亮点：Graph-Augmented RAG (图谱增强问答)
在最终的交互问答环节，本 Agent 摒弃了传统的“文档切片 -> 向量检索（Vector RAG）”模式，升级为**“图谱增强检索”**。
系统会将融合后的图谱拓扑结构（包含节点定义及 `prerequisite`、`contains` 等逻辑连线）实时转化为结构化上下文喂给大模型。当用户提问时，Agent 是沿着图谱的**“因果关联网络”**进行推理回答，而非单纯的文字片段拼接，彻底解决了医学问答中易产生的逻辑幻觉问题。
